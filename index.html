Websites based on the Pico CMS

Pico is a content management system (CMS). You can learn more about Pico at picocms.org.

Some appealing features of Pico:

  • Open-source: https://github.com/picocms/Pico
  • Small: 1.19 MB zipped
  • Secure
  • Very fast
  • Portable: no database, minimal PHP requirements

Why use Pico?

For a Webmaster or other content management professional, Pico's architecture makes it use very straightforward.

Templates in Twig

Twig is a template engine for PHP.

It allows the webmaster/developer to add placeholders for data in standard HTML files.

By using Twig Pico, can adopt the look and feel of virtually any commercially available template.

Content in Markdown

Markdown converts a simplified text format to standard HTML.

Pico stores the website's content in (markdown) text files.

The website' menu structure is (logically) based on the folder structure of these text files.

Pico gives direct access to data making editing, backup and transfer very easy and simple.

The routine work for a webmaster or other content manager is editing text and uploading files (mostly images).

One might expect a layman to need the administrative backend of more common database-driven CMSs however ... even they (e.g. administrative assistants) may appreciate the straightfoward process of simply typing their information into a text file and dropping an image into a folder as the totality of updating the website.

The text files an images can be reviewed, emailed and otherwise minipulated as normal.

Moving the website is simply copying the entire folder to a USB and uploading it somewhere else.

Other Features

Pico has a plugin architecture. Its functions can be extended with custom features.
